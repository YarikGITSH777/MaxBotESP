# MaxBotESP
Многофункциональная быстрая библиотека для Max бота на esp8266/esp32

Позволяет управлять устройствами (реле, светодиоды, датчики) через мессенджер Max, создавать меню с кнопками и обновлять прошивку по воздуху (OTA) прямо из чата.

## 📌 Возможности
**Легкий вес:** Не использует тяжелые JSON-библиотеки (ArduinoJson). Ручной потоковый парсинг экономит ~20кБ Flash и 5кБ RAM.

**Long Polling:** Устройство само опрашивает сервер (не нужен "белый IP" и порт 443).

**Inline-клавиатура:** Создание удобных меню с кнопками прямо в чате.

**OTA Обновления:** Присылайте .bin файл в чат, и плата сама обновится.

**Работа с файлами:** Бот умеет принимать файлы и картинки.

**Совместимость:** ESP8266 и ESP32.

## 📥 Установка
1. Нажмите Code -> Download ZIP.
2. В Arduino IDE: Скетч -> Подключить библиотеку -> Добавить .ZIP библиотеку.
3. Или скопируйте папку MaxBot в C:\Users\...\Documents\Arduino\libraries\.
   
## 🚀 Быстрый старт
````
#include <WiFi.h>
#include "MaxBot.h"

const char* ssid = "NAME";
const char* pass = "PASS";
const char* token = "ТОКЕН_ИЗ_BOTFATHER_MAX";

MaxBot bot(token);

void setup() {    
     WiFi.begin(ssid, pass);    
     while (WiFi.status() != WL_CONNECTED) 
        delay(500);
     }        
     configTime(3 * 3600, 0, "pool.ntp.org"); // Время нужно для HTTPS!        
     bot.attach(newMsg); // Подключаем обработчик
}

void loop() {    
  bot.tick(); // Обязательный тикер в loop
}

// Обработчик сообщений
void newMsg(MaxMsg& msg) {    
     if (msg.text == "Привет") {        
         bot.sendMessage("Привет! Я работаю на ESP32.", msg.chat_id);    
     }
}
````
## 📚 Документация
Структура сообщения (MaxMsg)
При входящем сообщении вызывается ваша функция, в которую передается структура MaxMsg:
````
struct MaxMsg {
    int64_t chat_id;      // ID чата (куда отвечать)
    int64_t user_id;      // ID отправителя
    String text;          // Текст сообщения
    String type;          // "message_created" или "message_callback"
    
    // Для кнопок
    String callback_data; // Данные нажатой кнопки
    String message_id;    // ID сообщения (для редактирования)

    // Для файлов
    bool isFile;          // true, если это файл
    String fileUrl;       // Ссылка на скачивание файла
    String fileName;      // Имя файла
};
````
## Основные методы
| Метод | Описание |
| ------ | ------- |
| `sendMessage(text)` | Отправить текст в последний активный чат |
| `sendMessage(text, chat_id)` | Отправить текст в конкретный чат |
| `sendMessage(text, chat_id, buttons)` | Отправить текст с кнопками |
| `answerCallback(msg_id, text)` | Ответить на нажатие кнопки (убрать "часики") |
| `updateFirmware(url)` | Обновить прошивку по ссылке (OTA) |

## Меню (Кнопки)
Для создания меню передайте строку с форматированием:

 \t — разделитель кнопок в одной строке.
 
 \n — перенос на новую строку.
````
String menu = "ВКЛ \t ВЫКЛ \n Статус";
bot.sendMessage("Выберите действие:", msg.chat_id, menu);
````
## 🔧 OTA Обновления
Библиотека умеет обновлять прошивку прямо из чата!

1. Скомпилируйте скетч (Скетч -> Экспорт бинарного файла).
2. Отправьте полученный .ino.bin файл боту в чат.
3. Плота скачает его, обновится и перезагрузится.
````
if (msg.isFile && msg.fileUrl.indexOf(".bin") != -1) {
    bot.updateFirmware(msg.fileUrl);
}
````
## ⚠️ Важные нюансы
1. HTTPS: Max работает только по HTTPS. Для ESP32 обязательно вызывайте configTime() в setup(), иначе соединение не установится.
2. Память: Библиотека использует ручной парсинг, что позволяет экономить память, но требует аккуратности при очень длинных сообщениях.
